Point Fixe & Zéro

Un point fixe de g est un x tel que g(x) = x. Un zéro de f est un x tel que f(x) = 0.

📖 Théorie
⚙️ Point Fixe
✂️ Dichotomie
📐 Newton
Définitions
Point Fixe : Soit g une fonction continue. On appelle point fixe de g tout réel x tel que :
g(x) = x
Point Zéro : On appelle zéro (ou racine) de f tout réel x tel que :
f(x) = 0
Lien : Trouver un zéro de f revient à trouver un point fixe de g, avec g(x) = x - f(x) ou toute autre forme équivalente.
Théorème du Point Fixe (Banach)
Si g est contractante sur [a,b] (∃k < 1 tel que |g'(x)| ≤ k) et g([a,b]) ⊂ [a,b], alors g admet un unique point fixe dans [a,b] et la suite xₙ₊₁ = g(xₙ) converge vers ce point fixe.
Condition : |g'(x)| < 1 sur l'intervalle
Algorithme Général — Point Fixe
Variables : x0, x1 : Réel ; eps : Réel ; n : Entier Début Lire(x0, eps) n ← 0 Répéter x1 ← g(x0) n ← n + 1 x0 ← x1 Jusqu'à |x1 - x0| < eps Écrire("Point fixe ≈ ", x1) Écrire("Itérations : ", n) Fin
Calcul — Méthode du Point Fixe
Entrez une fonction g(x) telle que g(x) = x au point fixe. Exemples :
g(x) = cos(x) g(x) = (x + 2/x) / 2 g(x) = x - (x*x - 2) / (2*x)
Le résultat s'affichera ici...
nxₙg(xₙ)|xₙ₊₁ - xₙ|
Méthode de Dichotomie
Principe : si f(a)·f(b) < 0, la fonction change de signe sur [a,b]. On divise l'intervalle par 2 à chaque étape.
m = (a + b) / 2 Si f(a)·f(m) < 0 → b = m, sinon a = m Arrêt quand (b - a) < ε
Le résultat s'affichera ici...
nabmf(m)
Méthode de Newton-Raphson
Convergence quadratique. Nécessite f et sa dérivée f'.
xₙ₊₁ = xₙ - f(xₙ) / f'(xₙ)
Le résultat s'affichera ici...
nxₙf(xₙ)xₙ₊₁|xₙ₊₁ - xₙ|

Méthode des Rectangles
On divise [a,b] en n sous-intervalles de largeur h = (b-a)/n.
Gauche : S = h × Σ f(a + i×h) pour i=0..n-1 Droite : S = h × Σ f(a + (i+1)×h) pour i=0..n-1 Milieu : S = h × Σ f(a + (i+½)×h) pour i=0..n-1

Algorithmes d'Optimisation

Trouver les extrema (maximum/minimum) d'une fonction par méthodes itératives.

✂️ Ternaire
∇ Gradient
📖 Théorie
Méthode de Recherche Ternaire
Pour une fonction unimodale sur [a,b], on divise en 3 parties :
m1 = a + (b-a)/3 m2 = b - (b-a)/3 Si f(m1) < f(m2) → b = m2 (minimum) Sinon → a = m1 Arrêt quand (b-a) < ε
Le résultat s'affichera ici...
Descente de Gradient
Algorithme itératif pour trouver un minimum local en suivant la pente négative :
xₙ₊₁ = xₙ - α × f'(xₙ) α = pas d'apprentissage (learning rate)
Converge si α est suffisamment petit.
Le résultat s'affichera ici...
nxₙf(xₙ)f'(xₙ)|Δx|
Théorie — Optimisation
Extremum : x* est un minimum local si f(x*) ≤ f(x) pour x voisin de x*.

Condition nécessaire (ordre 1) :
f'(x*) = 0 (point stationnaire)
Condition suffisante (ordre 2) :
f''(x*) > 0 → minimum local f''(x*) < 0 → maximum local f''(x*) = 0 → inflexion (indéterminé)
Comparaison des méthodes :
— Recherche ternaire : simple, unimodale uniquement, ne nécessite pas f'
— Descente de gradient : nécessite f', sensible au pas α
— Newton : quadratique, nécessite f' et f'', très rapide
Algorithme Ternaire — Pseudocode
Variables : a, b, m1, m2, eps : Réel Début Lire(a, b, eps) TantQue (b - a) > eps Faire m1 ← a + (b - a) / 3 m2 ← b - (b - a) / 3 Si f(m1) < f(m2) Alors b ← m2 Sinon a ← m1 FinSi FinTantQue Écrire("Minimum en x ≈ ", (a+b)/2) Fin

Tri à Bulles (Bubble Sort)
On compare deux éléments adjacents et on les permute s'ils sont dans le mauvais ordre. À chaque passage, le plus grand élément « remonte » à la fin.
Pour i de 0 à n-2 Faire Pour j de 0 à n-i-2 Faire Si T[j] > T[j+1] Alors permuter(T[j], T[j+1]) FinPour FinPour Complexité : O(n²)
Résultat...
Tri par Sélection (Selection Sort)
À chaque étape, on cherche le plus petit élément du sous-tableau non trié et on le place à sa position finale.
Pour i de 0 à n-2 Faire min ← i Pour j de i+1 à n-1 Faire Si T[j] < T[min] Alors min ← j FinPour permuter(T[i], T[min]) FinPour Complexité : O(n²) — minimum n-1 permutations
Résultat...
Tri par Insertion (Insertion Sort)
On insère chaque élément à sa bonne place dans la partie déjà triée. Très efficace pour les petits tableaux ou quasi-triés.
Pour i de 1 à n-1 Faire cle ← T[i] ; j ← i-1 TantQue j ≥ 0 et T[j] > cle Faire T[j+1] ← T[j] ; j ← j-1 FinTantQue T[j+1] ← cle FinPour Complexité : O(n²) au pire, O(n) au mieux

Recherche Séquentielle (Linéaire)
On parcourt le tableau du début à la fin et on s'arrête dès qu'on trouve la valeur. Fonctionne sur n'importe quel tableau (trié ou non).
i ← 0 TantQue i < n et T[i] ≠ x Faire i ← i + 1 FinTantQue Si i < n Alors trouvé en position i Sinon non trouvé Complexité : O(n)

Recherche Dichotomique (Binary Search)
Le tableau doit être trié. On compare avec l'élément du milieu, puis on continue à gauche ou à droite.
a ← 0 ; b ← n-1 TantQue a ≤ b Faire m ← (a + b) div 2 Si T[m] = x Alors trouvé en position m Sinon Si T[m] < x Alors a ← m+1 Sinon b ← m-1 FinTantQue Complexité : O(log₂ n)

Sur un grand tableau trié, la dichotomie est spectaculairement plus rapide.

La Récursivité

Une fonction qui s'appelle elle-même. Toujours composée d'un cas de base et d'un cas récursif.

n! Factorielle
🔢 Fibonacci
⚙ PGCD Euclide
🗼 Tours d'Hanoï
Factorielle Récursive
fact(0) = 1 ← cas de base fact(n) = n × fact(n-1) ← cas récursif
Profondeur d'appels = n. Pour n = 5 :
fact(5) = 5 × fact(4) = 5 × 4 × fact(3) = 5 × 4 × 3 × fact(2) = 5 × 4 × 3 × 2 × fact(1) = 5 × 4 × 3 × 2 × 1 × fact(0) = 120
Résultat...
Suite de Fibonacci
fib(0) = 0 fib(1) = 1 fib(n) = fib(n-1) + fib(n-2)
⚠ La version naïve récursive est en O(2ⁿ) — extrêmement lente. La version itérative (mémo) est en O(n).
Résultat...
PGCD — Algorithme d'Euclide
pgcd(a, 0) = a pgcd(a, b) = pgcd(b, a mod b)
Convergence très rapide : O(log min(a,b)).
Résultat...
Tours d'Hanoï
Déplacer n disques de la tour A vers la tour C en utilisant la tour B, sans jamais poser un grand sur un petit.
hanoi(n, A, B, C) : Si n = 1 : déplacer A → C Sinon : hanoi(n-1, A, C, B) déplacer A → C hanoi(n-1, B, A, C)
Nombre de déplacements = 2ⁿ - 1.
